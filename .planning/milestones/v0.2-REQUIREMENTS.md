## v1 Requirements

Requirements for milestone v0.2: Membros & Supabase Schema.

### Members (MEMB)

- [x] **MEMB-01**: Tabela `members` com campos completos (nome, birth_date, gender, marital_status, phone, email, endereço)
- [x] **MEMB-02**: Campos de batismo e conversão (baptism_date, conversion_date)
- [x] **MEMB-03**: Relacionamento com departments via `department_id`
- [x] **MEMB-04**: Status ativo/inativo com controle
- [x] **MEMB-05**: Timestamps padrão (created_at, updated_at, deleted_at soft delete)
- [x] **MEMB-06**: Avatar/photo via Supabase Storage

### Family Groups (FMLY)

- [x] **FMLY-01**: Tabela `family_groups` com nome do grupo/família
- [x] **FMLY-02**: Tabela `family_members` (relacionamento N:N entre members)
- [x] **FMLY-03**: Campo `relationship` (pai, mãe, filho, cônjuge, etc.)
- [x] **FMLY-04**: Líder do grupo familiar

### Member Timeline (TIME)

- [x] **TIME-01**: Tabela `member_timeline` com eventos ministeriais
- [x] **TIME-02**: Tipos: cargo, departamento, status, observação
- [x] **TIME-03**: Registros de alterações com user e timestamp

### Attendance (ATTD)

- [x] **ATTD-01**: Tabela `member_attendances` para controle de presença
- [x] **ATTD-02**: Vinculação a eventos
- [x] **ATTD-03**: Status: presente, ausente, justificar

### Roles (ROLE)

- [x] **ROLE-01**: Enum de cargos alinhado ao RBAC
- [x] **ROLE-02**: Tabela `member_roles` para histórico de cargos
- [x] **ROLE-03**: Cargos: Member, Leader, Treasurer, Admin, Super Admin

### Multi-Tenant & Security (SECU)

- [x] **SECU-01**: `church_id` em todas as tabelas
- [x] **SECU-02**: RLS policies por church_id
- [x] **SECU-03**: Índices para performance
- [x] **SECU-04**: Audit columns (created_by, updated_by)
